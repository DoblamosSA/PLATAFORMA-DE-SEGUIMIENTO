# Glosario

| Término | Significado en esta aplicación |
| --- | --- |
| Proyecto | Iniciativa de software, soporte o infraestructura que agrupa tareas y un equipo. |
| Tarea | Actividad asignable, asociada opcionalmente a un proyecto y controlada por SLA. |
| Subtarea | Desglose estimable de una tarea; aporta horas a la tarea principal. |
| SLA | Tiempo objetivo de resolución definido por tipo y prioridad. |
| Semáforo | Estado visual calculado de salud: planeado, vencido, en riesgo o saludable. |
| Tablero | Kanban por proyecto; sus columnas mapean a estados canónicos de tarea. |
| Responsable | Usuario líder de un proyecto. |
| Equipo | Usuarios asociados a un proyecto mediante `project_user`. |
| Rol primario | Rol global raíz del esquema organizacional, no eliminable. |
| Rol heredado | Rol de departamento que deriva permisos desde un rol padre. |
| Grant / deny | Otorgamiento o revocación explícita de un permiso en un rol. |
| Cumplida a tiempo | Indicador almacenado al cerrar una tarea según su fecha límite. |
